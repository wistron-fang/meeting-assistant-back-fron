# 前端上线 —— 服务器侧执行步骤（方案B：域名+HTTPS+同源反代）

> 域名：meeting.gigba-ai.cn ｜ ECS：47.76.244.232 ｜ 后端：127.0.0.1:8100
> 本地已备好：`deploy/dist.tar.gz`（前端静态包）、`deploy/nginx/meeting.conf`（nginx 配置）
> 在阿里云 **Workbench**（浏览器终端，root）里执行。

## 前置（用户侧，必须先完成，否则 HTTPS 签不了）
- [ ] 域名 `gigba-ai.cn` **实名认证已通过**（阿里云域名列表看「实名认证」列）。.cn 没实名不解析。
- [ ] 云解析加 **A 记录**：主机记录 `meeting`、类型 `A`、值 `47.76.244.232`、TTL 600。
- [ ] ECS 安全组放通 **80、443**（入方向）。
- [ ] 验证解析生效（本地或任意机器）：`nslookup meeting.gigba-ai.cn 223.5.5.5` 应返回 47.76.244.232。
      （现在还是 NXDOMAIN，加完记录等几分钟再验。）

> ⚠️ 解析没生效前，下面的 **B 段（certbot）会失败**。可以先做 A 段（HTTP 跑通），解析好了再做 B 段。

## A 段：上传 dist + 配 nginx（先 HTTP，80 端口）

### A1. 上传两个文件
- 用 Workbench 的「上传文件」把本地 `deploy/dist.tar.gz` 和 `deploy/nginx/meeting.conf` 传到 ECS（比如 `/root/`）。

### A2. 解压 dist 到站点目录
```bash
mkdir -p /opt/meeting/frontend/dist
tar xzf /root/dist.tar.gz -C /opt/meeting/frontend/dist
ls /opt/meeting/frontend/dist        # 应看到 index.html assets/ 等
```

### A3. 装 nginx + 放配置
```bash
yum install -y nginx || dnf install -y nginx
cp /root/meeting.conf /etc/nginx/conf.d/meeting.conf
nginx -t                              # 语法检查，必须 ok
systemctl enable --now nginx
systemctl reload nginx
```

### A4. 验证 HTTP（解析没生效也能先用 IP 测）
```bash
curl -I http://127.0.0.1/             # 应 200，返回 index.html
curl -I http://127.0.0.1/auth/        # 反代到后端，应有响应（非 nginx 404 即可）
```
- 浏览器开 `http://meeting.gigba-ai.cn`（需解析已生效）能看到登录页即 A 段成功。

---

## B 段：上 HTTPS（解析生效后再做）

```bash
yum install -y certbot python3-certbot-nginx || dnf install -y certbot python3-certbot-nginx
certbot --nginx -d meeting.gigba-ai.cn
# 按提示填邮箱、同意条款；问是否强制 https 跳转选 “是(2)”
systemctl reload nginx
```
- certbot 会自动改 `/etc/nginx/conf.d/meeting.conf`，加 `listen 443 ssl` + 证书路径 + 80→443 跳转。
- 证书自动续期已由 certbot 定时任务接管，无需手动。

### B 验证
```bash
curl -I https://meeting.gigba-ai.cn/
```
浏览器开 `https://meeting.gigba-ai.cn`：
- [ ] 能加载界面（小锁🔒，无证书告警）
- [ ] 能登录（用 `admin_create_user.py` 建的账号；登录页已无注册入口）
- [ ] 能提交任务、能下载产物（md/pdf 文件名正常 = Content-Disposition 透传 OK）
- [ ] 刷新子路由页面不 404（SPA 兜底 try_files 生效）
- [ ] 提交大音频/文档不报 413（client_max_body_size 200m 生效）

## 回滚 / 常见问题
- nginx 起不来：`nginx -t` 看报错；`journalctl -u nginx -n 50`。
- 80 端口被占：`ss -ltnp | grep :80`。
- 反代 502：后端没起？`systemctl status meeting-api`，确认 8100 在听 `curl -I http://127.0.0.1:8100/hello`。
- 改了前端要更新：本地重新 `npm run build` → 重打 `dist.tar.gz` → 覆盖解压 → `systemctl reload nginx`（静态文件无需 reload，但反代配置变了才需要）。
