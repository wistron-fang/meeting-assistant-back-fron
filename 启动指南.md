# 会议纪要助手 - 启动指南

> 适用环境: Windows 10/11 + Docker Desktop + Conda + Node.js
> 关于上生产环境的差异,见 [DEPLOYMENT.md](./DEPLOYMENT.md)

---

## 一、首次部署 (新电脑只做一次)

### 1.1 装 3 个前置工具

| 工具 | 下载 | 用途 |
|---|---|---|
| Docker Desktop | https://www.docker.com/products/docker-desktop | 跑 PostgreSQL + Redis |
| Miniconda / Anaconda | https://docs.conda.io/en/latest/miniconda.html | Python 环境隔离 |
| Node.js (LTS ≥ 18) | https://nodejs.org/ | 前端 Vite 开发服务器 |

装完都打开一次确认能用:

```powershell
docker --version
conda --version
node --version
npm --version
```

### 1.2 创建后端 conda 环境

项目里 `backend/start_all.bat` 默认 conda env 路径是 `D:\conda\envs\multiagent`。
如果你的 conda 装在别处,需要改 `start_all.bat` 里第 18 行的 `CONDA_ENV=` 路径。

```powershell
conda create -n multiagent python=3.11 -y
conda activate multiagent
cd E:\meeting_minutes_assistant\backend
pip install -r requirements.txt
```

> requirements.txt 体量很大 (LLM + 文档解析 + 向量库),首次装 10-20 分钟很正常。

### 1.3 装前端依赖

```powershell
cd E:\meeting_minutes_assistant\frontend
npm install
```

### 1.4 起 PostgreSQL + Redis 容器

```powershell
cd E:\meeting_minutes_assistant
docker compose up -d
```

第一次会拉镜像 (postgres:15-alpine + redis:7-alpine,共约 250 MB)。
拉完后 `docker ps` 应该能看到 `meeting_postgres` 和 `meeting_redis` 都是 healthy 状态。

### 1.5 配置 .env

```powershell
cd E:\meeting_minutes_assistant\backend
copy .env.example .env
notepad .env
```

把以下 key 填上真实值 (其它可选 key 默认空也能跑):

| Key | 说明 |
|---|---|
| `DASHSCOPE_API_KEY` | 阿里云百炼,LLM 主调用,**必填** |
| `BOCHA_API_KEY` | 博查搜索,**必填** |
| `ALIYUN_ACCESS_KEY_ID/SECRET` | 阿里云 RAM AK,**录音转写需要** |
| `ALIYUN_NLS_APPKEY` | 智能语音控制台 AppKey,**录音转写需要** |
| `ALIYUN_OSS_ENDPOINT/BUCKET` | OSS 桶 (大文件 ASR 中转用),**录音转写需要** |
| `JWT_SECRET_KEY` | 随便填长串字符,生产环境必须重换 |

### 1.6 初始化数据库表

第一次启动后端 (见下文 §二),代码会自动 `Base.metadata.create_all` 建表。
如果之后改了模型字段,需要手动 ALTER TABLE,比如最近加的录音转写 3 个字段:

```powershell
docker exec meeting_postgres psql -U postgres -d meeting_assistant -c "ALTER TABLE meeting_tasks ADD COLUMN IF NOT EXISTS audio_path VARCHAR(500); ALTER TABLE meeting_tasks ADD COLUMN IF NOT EXISTS audio_filename VARCHAR(255); ALTER TABLE meeting_tasks ADD COLUMN IF NOT EXISTS transcript_path VARCHAR(500);"
```

## 二、日常启动 (每次开发都做)

### 2.1 起 docker (如果没在跑)

```powershell
docker compose -f E:\meeting_minutes_assistant\docker-compose.yml up -d
```

或者直接打开 Docker Desktop 在 Containers 页点启动。

### 2.2 起后端 (3 个窗口)

```powershell
cd E:\meeting_minutes_assistant\backend
.\start_all.bat
```

会自动弹 **3 个 cmd 黑窗口**:

| 窗口标题 | 服务 |
|---|---|
| `uvicorn FastAPI 8100` | API 服务,监听 :8100 |
| `celery worker` | 跑会议纪要异步任务 |
| `celery beat` | 触发每日清理 |

> Windows 上 celery 禁用了 `-B` 参数,所以 worker / beat 必须拆两个进程,这与生产部署要求一致。

### 2.3 起前端

```powershell
cd E:\meeting_minutes_assistant\frontend
npm run dev
```

看到 `Local: http://localhost:5273/` 就 OK。

### 2.4 打开浏览器

访问 **http://localhost:5273**,登录后到 "会议纪要整理" 页。

## 三、日常关闭

按这个顺序优雅关掉:

1. 前端窗口 `Ctrl+C` 退出 `npm run dev`
2. 关掉 3 个后端黑窗口 (×)
3. 想完全停掉 docker:`docker compose -f E:\meeting_minutes_assistant\docker-compose.yml stop`
   - 只 stop 不 down,数据保留,下次 `start` 秒起
   - 用 `down` 会删容器但 volume 保留,数据仍在
   - 用 `down -v` 会**连数据库一起删**,慎用

## 四、验证清单 (出问题先按这个排查)

| 检查 | 命令 | 期望 |
|---|---|---|
| FastAPI 通 | `curl http://localhost:8100/hello` | 返回 hello world JSON |
| PG 通 | `docker exec meeting_postgres pg_isready -U postgres` | `accepting connections` |
| Redis 通 | `docker exec meeting_redis redis-cli ping` | `PONG` |
| Celery 起 | celery worker 窗口看到 `ready.` | 无报错 |
| 前端起 | 浏览器开 :5273 | 看到登录页 |

常见坑:

- **Celery 接到任务就报 "database does not exist"** → `.env` 里 `POSTGRES_PORT` 应该是 `5532`,不是默认的 `5432` (host:5532 → container:5432)
- **录音上传后报 "环境变量未配置"** → `.env` 里 5 个 `ALIYUN_*` 没填全,或者改完 .env 没重启 celery worker
- **`start_all.bat` 报 uvicorn.exe / celery.exe not found** → `CONDA_ENV` 路径不对,改 bat 里的路径

## 五、迁移到另一台 Windows 电脑

完全可以打包。两台机器都没装过这个项目时,**省事顺序**是:

### 方案 A: 目标机器联网 (推荐)

只需要拷贝**项目源码**,目标机器自己拉镜像 + 装依赖:

1. 把整个 `meeting_minutes_assistant/` 目录打包 (排除 `node_modules`、`.venv`、`backend/storage/`、`data/`、`__pycache__/`)
   ```powershell
   # 在源机器
   cd E:\
   tar -czf meeting_minutes_assistant.tar.gz `
       --exclude=meeting_minutes_assistant/frontend/node_modules `
       --exclude=meeting_minutes_assistant/data `
       --exclude=meeting_minutes_assistant/backend/storage `
       --exclude="**/__pycache__" `
       meeting_minutes_assistant
   ```
2. 目标机器解压到任意位置 (比如 `D:\meeting_minutes_assistant\`)
3. 按上文 **§一 首次部署** 全套走一遍

> 包大小约 1-5 MB (代码本身很小)。

### 方案 B: 目标机器**没网**,镜像也要离线带过去

适用于:目标机器内网隔离,无法拉 docker hub。

#### 步骤 1: 在**源机器**导出 docker 镜像

```powershell
cd E:\meeting_minutes_assistant
docker save -o meeting-images.tar postgres:15-alpine redis:7-alpine
```

生成 `meeting-images.tar` 大约 **200 MB**。

#### 步骤 2: (可选) 导出数据库当前数据

如果想把现有的会议任务一起搬过去,导出 postgres 数据卷:

```powershell
# 先停容器,避免写盘冲突
docker compose stop postgres

# 把 volume 打成 tar (注意 volume 名前缀是 compose 项目名,这里是 meeting_minutes_assistant)
docker run --rm `
    -v meeting_minutes_assistant_postgres_data:/data `
    -v ${PWD}:/backup `
    alpine tar czf /backup/pg-data.tar.gz -C /data .

docker compose start postgres
```

#### 步骤 3: (可选) 导出 conda 环境

最稳的方式是 `conda env export`,目标机器有网就能复现:

```powershell
conda activate multiagent
conda env export --no-builds > backend\environment.yml
```

如果目标机器也没网,改用 `conda-pack`:

```powershell
conda install -n base conda-pack
conda pack -n multiagent -o multiagent-env.tar.gz
```

`multiagent-env.tar.gz` 大约 **2-4 GB** (Python + 所有 pip 包都打包了)。

#### 步骤 4: 打包带走

用 U 盘 / 网盘把这几样东西拷到目标机器:

- 项目代码 `.tar.gz` (方案 A 第 1 步那个)
- `meeting-images.tar` (docker 镜像)
- `pg-data.tar.gz` (可选,数据库内容)
- `environment.yml` 或 `multiagent-env.tar.gz` (二选一,conda 环境)
- `backend\.env` (敏感,单独存放,不要传公共网盘)

#### 步骤 5: 在**目标机器**恢复

```powershell
# 1. 装 Docker Desktop / Conda / Node.js (前置工具,见 §1.1)

# 2. 解压项目代码到 D:\meeting_minutes_assistant\

# 3. 加载 docker 镜像
docker load -i meeting-images.tar
docker images   # 应该看到 postgres:15-alpine 和 redis:7-alpine

# 4. 起 docker (这步会用上面 load 进来的镜像,不再拉网络)
cd D:\meeting_minutes_assistant
docker compose up -d

# 5. (可选) 恢复数据库数据
docker compose stop postgres
docker run --rm `
    -v meeting_minutes_assistant_postgres_data:/data `
    -v ${PWD}:/backup `
    alpine sh -c "cd /data && tar xzf /backup/pg-data.tar.gz"
docker compose start postgres

# 6. 恢复 conda 环境 (二选一)
#   方式 A: 有网
conda env create -f backend\environment.yml

#   方式 B: 用 conda-pack
mkdir D:\conda\envs\multiagent
tar -xzf multiagent-env.tar.gz -C D:\conda\envs\multiagent
D:\conda\envs\multiagent\Scripts\conda-unpack.exe

# 7. 装前端依赖 (有网的话直接 npm install,没网需要先把 node_modules 也打包带过来,或起一个内网 npm registry)
cd frontend
npm install

# 8. 把 .env 放到 backend\.env

# 9. 按 §二 日常启动 跑起来
```

#### 步骤 6: 如果改了硬盘盘符路径

`start_all.bat` 里 `CONDA_ENV=D:\conda\envs\multiagent` 是写死的,如果目标机器 conda 在 `C:\` 等其他位置,**记得改这一行**。

## 六、文件位置速查

| 文件 | 作用 |
|---|---|
| `backend/.env` | 全部 API Key 和密码,**不要进 git** |
| `backend/start_all.bat` | Windows 一键启 3 个后端进程 |
| `backend/app/service/meeting/asr.py` | 语音识别 (极速版 + Qwen 普通版) |
| `backend/app/service/meeting/tasks.py` | Celery 任务主流程 |
| `docker-compose.yml` | PG + Redis 容器定义 |
| `data/meeting/{user_id}/{task_id}/` | 单任务产物 (md / pdf / transcript / 源音频) |
| `frontend/vite.config.ts` | 前端端口 (5273) 和后端代理 |
